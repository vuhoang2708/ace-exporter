Tài Liệu Hướng Dẫn & Cấu Hình ACE Exporter

Chào mừng bạn đến với tài liệu hướng dẫn vận hành hệ thống xuất log hội thoại **ACE Exporter** dành cho Codex, Claude Code và AWS Kiro.

1. Tổng Quan Về Hệ Thống

**ACE Exporter** (Antigravity Chat Exporter) là một phần mở rộng (`VS Code Extension`) chuyên dụng được thiết kế để giải quyết nhu cầu lưu trữ, phân tích và kiểm toán (`audit`) các phiên đối thoại giữa nhà phát triển và các trợ lý AI.

Thay vì lưu trữ dưới dạng nhật ký thô (`JSONL`/`JSON`), hệ thống tự động phát hiện, trích xuất cấu trúc và biên dịch thông minh sang tài liệu `Markdown` chất lượng cao, giúp định dạng dễ đọc và lưu giữ dấu vết mã nguồn tiện lợi.

🤖 Claude Code Support

Hỗ trợ đầy đủ các dự án Claude Code bằng cách phân tích tập tin log trong thư mục `.claude/projects`. Tách biệt tin nhắn văn bản và các sự kiện gọi công cụ.

⚡ OpenAI Codex Support

Khả năng phân tích sâu dòng nhật ký Codex, lưu vết các hàm gọi (`function_call`), dữ liệu lập trình viên, và lời bình của trợ lý AI.

🪄 AWS Kiro Support

Tương thích hoàn hảo với cấu trúc log của Kiro thông qua cả kiểu bao dữ liệu phẳng và phân cấp lồng nhau.

2. Hướng Dẫn Cài Đặt

Để cài đặt và chạy thử nghiệm phần mở rộng trên môi trường VS Code cục bộ của bạn, vui lòng làm theo các bước dưới đây:

Terminal - Cài đặt dependencies và đóng gói 
# 1. Cài đặt các thư viện phụ thuộc (node_modules)
npm install

# 2. Biên dịch TypeScript sang JavaScript
npm run compile

# 3. Đóng gói mã nguồn thành file bundle duy nhất bằng esbuild
npm run bundle

Sau khi đóng gói thành công, bạn có thể bấm phím F5 trong VS Code. Một cửa sổ VS Code mới (Extension Development Host) sẽ khởi chạy với tính năng ACE Exporter đã được kích hoạt hoàn toàn.

3. Tính Năng Xuất & Mô Phỏng Lọc Dữ Liệu

ACE Exporter cung cấp hai chế độ xuất nhật ký chính nhằm phục vụ các mục đích sử dụng khác nhau:

  • Chế độ Clean (Mặc định): Chỉ xuất các tin nhắn đối thoại thuần túy của Người dùng (`User`) và Trợ lý AI (`Assistant`), phù hợp cho việc đọc hiểu nhanh.
  • Chế độ Audit (Kiểm toán): Giữ lại toàn bộ dữ liệu gồm các chỉ dẫn hệ thống (`system`/`developer`), các lượt AI tự động gọi lệnh/chạy tool (`tool_use` / `tool_result`), và các bình luận nội bộ để nhà phát triển theo dõi chi tiết luồng xử lý lỗi của AI.

4. Các Câu Lệnh & Phím Tắt Khởi Chạy

Mở bảng lệnh của VS Code bằng tổ hợp phím Ctrl+Shift+P (Windows) hoặc Cmd+Shift+P (macOS) và chọn các lệnh sau:

Tên lệnh (Command Title) Mã Lệnh (VS Code Command) Chức năng chi tiết
Export Chat Session... `ace.exportChat` Khởi động luồng chọn thủ công: Nguồn log -> Tệp log trong lịch sử -> Chế độ Clean/Audit.
Export Latest Session (Quick) `ace.exportLatest` Xuất nhanh phiên làm việc mới nhất dựa trên các cài đặt mặc định (phím tắt trên thanh trạng thái).
Open Output Folder `ace.openOutputFolder` Mở trực tiếp thư mục tự động lưu trữ file Markdown của bạn bằng trình duyệt file hệ thống.
Show Status `ace.showStatus` Hiển thị trạng thái các thiết lập mặc định hiện tại lên bảng điều khiển Output của VS Code.

6. Khắc Phục Các Sự Cố Thường Gặp

            
Dưới đây là một số vấn đề thường gặp và cách tự giải quyết nhanh:

            
    
                
  • Lỗi: "No session logs found": Đảm bảo các tool AI tương ứng đã chạy và ghi log. Hãy kiểm tra các thư mục mặc định có tồn tại tập tin `.json`/`.jsonl` nào không:
                  
    - Claude Code: `~/.claude/projects`
                  
    - OpenAI Codex: `~/.codex/sessions`
                  
    - AWS Kiro: `~/.kiro/sessions`
                
    • 
                
  • File xuất bị cắt bớt dữ liệu Tool Output: Điều này do tham số `toolOutputMaxChars` giới hạn ký tự (mặc định 2000) để tối ưu hóa hiệu năng mở file của VS Code. Bạn có thể tăng tham số này lên trong phần cài đặt nếu muốn xem toàn bộ.
    • 
                
  • Auto-save không hoạt động: Đảm bảo bạn đã tick chọn `autoSave: true` và nhập đúng đường dẫn thư mục tồn tại trên ổ cứng tại cấu hình `outputFolder`.
